Arv

En klass kan ärva egenskaper och metoder från en annan klass. Detta gör att du kan arbeta i mindre moduler och kan göra att man utvecklar sin egen kod.

        
            class Klass1{
                int attribut;

                public void metod(){
                    attribut = 3;   // Fungerar bra.
                }
            }
            class Klass2 extends Klass1{

            }

            Klass2 k2 = new Klass2();
            k2.attribut; // Fungerar mycket bra.
            k2.metod(); // Fungerar bra

Private

Om du sätter ett attribut eller metod till private kommer du bara åt dessa via den egna klassen eller en instans av klassen.

        
            class Klass1{
                private int attribut;

                public void metod(){
                    attribut = 3;   // Fungerar bra då vi är i klassens instans. 
                }
            }
            class Klass2 extends Klass1{

            }

            Klass1 k1 = new Klass1();
            Klass2 k2 = new Klass2();
            k2.attribut; // Fungerar inta alls.
            k1.attribut; // Fungerar inte heller för vi är utanför klassen.
            k1.metod();  // Fungerar bra
            k2.metod();  // Fungerar bra.. Men gör kanske inte vad du vill

Protected

Protected är en egenskap som gör att attributet eller metoden är nåbar i ett arv. I Java betyder även detta att så länge du är i samma paket kommer du åt attributen eller metoderna.

        
            class Klass1{
                protected int attribut;

                public void metod(){
                    attribut = 3;   // Fungerar bra då vi är i klassens instans. 
                }
            }
            class Klass2 extends Klass1{

            }
            Klass1 k1 = new Klass1();
            Klass2 k2 = new Klass2();
            k2.attribut; // Fungerar eftersom vi är i paketet.
            k1.attribut; // Fungerar eftersom vi är i paketet.
            k1.metod();  // Fungerar bra
            k2.metod();  // Fungerar bra.. Men gör kanske inte vad du vill

Polymorfism

Att göra överlagringar är ett ganska vanligt sätt att göra fler ingångar.

Det reserverade ordet för att markera att en metod är @Override.

        
            class Klass1{
                protected int attribut;

                public void metod(){
                    attribut = 3;   // Fungerar bra då vi är i klassens instans. 
                }
            }
            class Klass2 extends Klass1{

                @Override
                public void metod(){
                    attribut = 4;   // Fungerar bra då vi är i klassens instans. 
                }

            }
            Klass1 k1 = new Klass1();
            Klass2 k2 = new Klass2();
            k1.metod();  // Kör metod i Klass1
            k2.metod();  // Kör metid i Klass2
